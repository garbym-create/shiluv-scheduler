# מערכת שיבוץ שילוב

לוח בקרה לשיבוץ אוטומטי של טיפולים שבועיים לתלמידי שילוב. המערכת מקבלת את שעות העבודה של המטפלים, השעות הפנויות של התלמידים והצרכים של כל אחד — ומחשבת את השיבוץ שמקסם את מספר הטיפולים שניתן לקיים, תוך שמירה על כל האילוצים.

## מה המערכת עושה

- **התאמת צרכים** — לכל צורך של תלמיד נבחרים רק המטפלים שמספקים בדיוק את סוג הטיפול הזה.
- **הצלבת חלונות זמן** — משבצת רק בשעות שבהן גם התלמיד וגם המטפל פנויים.
- **אכיפת אילוצים** — תלמיד לא בשני טיפולים באותה שעה, גודל הקבוצה של המטפל נשמר, ומטפל נותן טיפול אחד בלבד בכל שעה.
- **חיפוש אופטימלי** — אלגוריתם Backtracking עם היוריסטיקת MRV ו-branch & bound, שמוצא את המקסימום שניתן לשבץ (ולא "קרוב").
- **שתי תצוגות ללוח** — רשת שבועית מלאה (ימים × שעות) או רשימה לפי יום, לבחירתכם.
- **הורדת הלוח** — ייצוא לוח מעוצב ל**אקסל** (`.xls`), ל**וורד** (`.doc`, להדפסה), או ל-CSV גולמי.
- **שמירה וטעינה** — שמירת כל העבודה (מטפלים, תלמידים, שעות) לקובץ במחשב, וטעינתה בחזרה בכל עת. הקובץ נשאר אצלכם — ללא שרת.

## הפעלה — הדרך הפשוטה (ללא התקנות)

הקובץ **`index.html`** הוא עצמאי לחלוטין: לוחצים עליו פעמיים והמערכת נפתחת בדפדפן.
אין צורך להתקין שום דבר. אפשר גם להעלות אותו כמו שהוא ל-GitHub Pages ולשתף בקישור.

> טעינה ראשונה דורשת חיבור לאינטרנט (המערכת מושכת את React ואת הגופנים מרשת);
> לאחר מכן היא עובדת מהר, וכל הנתונים נשארים מקומית בדפדפן.

מסך הפתיחה מסביר על המערכת ועל שמירת הפרטיות, ובלחיצה על **"כניסה למערכת"** נכנסים ללוח הבקרה.

## הפעלה — למפתחים (Vite)

מי שרוצה לפתח מקומית עם שרת פיתוח יכול להשתמש ב-[Node.js](https://nodejs.org/) (גרסה 18+):

```bash
npm install
npm run dev
```

לאחר עריכת `src/App.jsx`, בונים מחדש את הקובץ העצמאי עם:

```bash
bash build-standalone.sh
```

## מבנה הפרויקט

```
index.html            — האפליקציה העצמאית המוכנה לשימוש (נבנית אוטומטית מ-App.jsx)
build-standalone.sh   — בונה את index.html מתוך src/App.jsx
serve.ps1             — שרת סטטי קטן לתצוגה מקומית (PowerShell, ללא Node)
src/
  App.jsx             — קוד המקור: ממשק + מנוע השיבוץ + מסך הפתיחה
  main.jsx            — נקודת כניסה ל-React (מסלול ה-Vite)
```

## פרטיות

המערכת מיועדת לתכנון פנימי בלבד. מומלץ לזהות כל תלמיד בכינוי או בראשי תיבות — לא בשם המלא האמיתי. כל הנתונים נשמרים בזיכרון הדפדפן בלבד ואינם נשלחים לשום שרת.
